# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in ExcaliHub, please follow these steps:

### Do NOT:
- Open a public issue on GitHub
- Discuss the vulnerability in public forums

### DO:
1. **Open a private security advisory** on GitHub:
   - Go to the repository
   - Click **Security** tab
   - Click **Report a vulnerability**
   
2. **OR** contact the maintainer directly:
   - Open an issue marked as `[PRIVATE SECURITY]` (will be visible only to maintainers)

## What to Include

When reporting a vulnerability, please include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if you have one)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 1 week
- **Fix and disclosure**: Within 30 days (depending on severity)

## Security Considerations

This extension handles sensitive data including:

- GitHub OAuth tokens
- User authentication data
- Access to private repositories

We take security seriously and appreciate responsible disclosure.

## Supported Versions

Only the latest version receives security updates. Please ensure you're using the most recent version.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
